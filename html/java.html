Variables y Tipos de Datos

Java es un lenguaje fuertemente tipado - cada variable debe declarar su tipo

Declaración de Variables

En Java, debes especificar el tipo de dato antes del nombre de la variable. Esto ayuda a prevenir errores y hace el código más seguro.

// Declaración básica: tipo nombreVariable = valor;
int edad = 25;
String nombre = "Carlos";
double salario = 3500.50;
boolean esEstudiante = true;

// Puedes declarar sin inicializar
int numeroArticulos;
numeroArticulos = 10;  // Asignar valor después

// Constantes (no pueden cambiar)
final double PI = 3.14159;
final String EMPRESA = "Tech Corp";

Tipos de Datos Primitivos

Java tiene 8 tipos primitivos que son la base de todos los datos:

// Números enteros
byte pequeño = 127;        // -128 a 127
short corto = 32000;       // -32,768 a 32,767
int entero = 2147483647;   // Más común (±2.1 billones)
long largo = 9223372036L;  // Números muy grandes (nota la L)

// Números decimales
float decimal = 3.14f;     // 6-7 dígitos (nota la f)
double doble = 3.14159;    // 15 dígitos (más preciso)

// Otros
char letra = 'A';          // Un solo carácter
boolean verdadero = true;  // true o false

Tipos de Referencia

// String - cadenas de texto
String mensaje = "Hola Java!";
String vacio = "";  // String vacío válido

// Arrays - colecciones de elementos
int[] numeros = {1, 2, 3, 4, 5};
String[] nombres = new String[3];  // Array de 3 elementos
nombres[0] = "Ana";
nombres[1] = "Luis";
nombres[2] = "María";

// Clases envolventes (Wrapper)
Integer numeroObjeto = 100;  // Versión objeto de int
Double decimalObjeto = 3.14; // Versión objeto de double
💡 Consejo: Usa int para números enteros y double para decimales en la mayoría de casos. Son los más eficientes y comunes.
⚠️ Importante: Java es case-sensitive. 'nombre', 'Nombre' y 'NOMBRE' son variables diferentes.

Clases y Objetos

Java es un lenguaje orientado a objetos - todo gira en torno a clases y objetos

Definiendo una Clase

Una clase es como un plano o plantilla para crear objetos. Define atributos (datos) y métodos (comportamientos).

// Definición de una clase simple
public class Persona {
    // Atributos (propiedades)
    private String nombre;
    private int edad;
    private String email;
    
    // Constructor
    public Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }
    
    // Métodos (comportamientos)
    public void saludar() {
        System.out.println("Hola, soy " + nombre);
    }
    
    public void cumplirAños() {
        edad++;
        System.out.println("¡Feliz cumpleaños! Ahora tengo " + edad);
    }
    
    // Getters y Setters
    public String getNombre() {
        return nombre;
    }
    
    public void setEmail(String email) {
        this.email = email;
    }
}

Creando y Usando Objetos

// Crear objetos (instancias) de la clase
Persona persona1 = new Persona("Ana", 25);
Persona persona2 = new Persona("Carlos", 30);

// Usar los métodos del objeto
persona1.saludar();        // Imprime: Hola, soy Ana
persona1.cumplirAños();    // Imprime: ¡Feliz cumpleaños! Ahora tengo 26

// Acceder a propiedades mediante getters/setters
String nombre = persona1.getNombre();
persona1.setEmail("ana@email.com");

// Cada objeto es independiente
persona2.saludar();        // Imprime: Hola, soy Carlos

Principios Básicos de POO

  • Encapsulación: Usar private para proteger datos y public para métodos de acceso
  • Abstracción: Mostrar solo lo necesario, ocultar la complejidad interna
  • Herencia: Las clases pueden heredar características de otras clases
  • Polimorfismo: Los objetos pueden tomar múltiples formas
💡 Consejo: Siempre usa private para los atributos y proporciona métodos públicos (getters/setters) para acceder a ellos. Esto se llama encapsulación.

Métodos

Los métodos definen las acciones que pueden realizar los objetos

Estructura de un Método

Un método tiene modificadores de acceso, tipo de retorno, nombre, parámetros y cuerpo.

// Estructura: modificador tipoRetorno nombreMetodo(parametros) { cuerpo }

// Método simple sin retorno
public void imprimirMensaje() {
    System.out.println("¡Hola desde el método!");
}

// Método con parámetros
public void saludar(String nombre) {
    System.out.println("Hola, " + nombre);
}

// Método que retorna un valor
public int sumar(int a, int b) {
    return a + b;
}

// Método con múltiples parámetros
public double calcularPromedio(double nota1, double nota2, double nota3) {
    double suma = nota1 + nota2 + nota3;
    return suma / 3;
}

Tipos de Métodos

public class Calculadora {
    // Método estático (no necesita objeto)
    public static int multiplicar(int x, int y) {
        return x * y;
    }
    
    // Método de instancia
    private double resultado;
    
    public void guardarResultado(double valor) {
        this.resultado = valor;
    }
    
    // Método sobrecargado (mismo nombre, diferentes parámetros)
    public int sumar(int a, int b) {
        return a + b;
    }
    
    public double sumar(double a, double b) {
        return a + b;
    }
    
    public int sumar(int a, int b, int c) {
        return a + b + c;
    }
}

Usando Métodos

// Método main - punto de entrada del programa
public static void main(String[] args) {
    // Llamar método estático
    int producto = Calculadora.multiplicar(5, 3);  // 15
    
    // Crear objeto y llamar métodos de instancia
    Calculadora calc = new Calculadora();
    int suma1 = calc.sumar(10, 20);       // 30
    double suma2 = calc.sumar(3.5, 2.5);  // 6.0
    int suma3 = calc.sumar(1, 2, 3);      // 6
    
    calc.guardarResultado(suma1);
}
💡 Consejo: Los nombres de métodos deben empezar con minúscula y usar camelCase: calcularTotal(), obtenerNombre(), esValido().

Control de Flujo

Controla cómo y cuándo se ejecuta tu código con condicionales y bucles

Condicionales (if/else)

// if simple
int edad = 18;
if (edad >= 18) {
    System.out.println("Eres mayor de edad");
}

// if-else
double nota = 7.5;
if (nota >= 6.0) {
    System.out.println("Aprobado");
} else {
    System.out.println("Reprobado");
}

// if-else if-else
int hora = 14;
if (hora < 12) {
    System.out.println("Buenos días");
} else if (hora < 19) {
    System.out.println("Buenas tardes");
} else {
    System.out.println("Buenas noches");
}

// Operador ternario (if corto)
String mensaje = (edad >= 18) ? "Mayor" : "Menor";
System.out.println(mensaje);

Switch

// Switch tradicional
int dia = 3;
String nombreDia;

switch (dia) {
    case 1:
        nombreDia = "Lunes";
        break;
    case 2:
        nombreDia = "Martes";
        break;
    case 3:
        nombreDia = "Miércoles";
        break;
    default:
        nombreDia = "Día inválido";
        break;
}

// Switch mejorado (Java 14+)
String tipo = switch (dia) {
    case 1, 2, 3, 4, 5 -> "Día laboral";
    case 6, 7 -> "Fin de semana";
    default -> "Día inválido";
};

Bucles

// Bucle for tradicional
for (int i = 0; i < 5; i++) {
    System.out.println("Iteración: " + i);
}

// For-each (para colecciones)
String[] frutas = {"Manzana", "Pera", "Uva"};
for (String fruta : frutas) {
    System.out.println("Fruta: " + fruta);
}

// Bucle while
int contador = 0;
while (contador < 5) {
    System.out.println("Contador: " + contador);
    contador++;
}

// Do-while (ejecuta al menos una vez)
int numero;
do {
    numero = (int)(Math.random() * 10);
    System.out.println("Número: " + numero);
} while (numero != 7);

// Control de bucles
for (int i = 0; i < 10; i++) {
    if (i == 3) continue;  // Salta a la siguiente iteración
    if (i == 7) break;     // Sale del bucle
    System.out.println(i);
}
💡 Consejo: Usa for cuando sepas el número de iteraciones, while cuando dependas de una condición, y for-each para recorrer colecciones.
⚠️ Importante: Siempre incluye 'break' en cada case del switch (excepto en el switch mejorado) para evitar que se ejecuten los casos siguientes.
